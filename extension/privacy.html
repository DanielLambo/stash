Privacy Policy — Stash

Last updated: 26 April 2026
No analytics No telemetry No data sale No remote code Local-first No tracking

The short version

Stash runs on your device. Nothing is sent to the developer of Stash. There is no Stash server. Stash makes no third-party network requests of any kind — no analytics, telemetry, error reporting, font CDNs, or favicon fetches. The only outgoing requests are to a sync server you choose, only when you opt in.

What Stash reads

Things you copy

When you press Cmd+C / Ctrl+C, or right-click → Copy on a web page, Stash records the result so you can paste it later. Capture is event-driven; we do not poll.

The system clipboard, on demand

When you open the Stash popup or click its manual capture button, Stash reads the system clipboard one time so it can record copies you made in another app or on a restricted page.

Trigger detection in an editable field — only if you turn on snippets

Snippet expansion is a productivity feature that replaces a short trigger you type (like ;sig) with an expansion you've written yourself. It is off by default. When you enable it, Stash detects user-defined trigger sequences in the editable field you're currently typing into and keeps a small in-memory window of the most recent characters — sized to your longest trigger. It compares the window's tail against the triggers you defined. On a match, it replaces the trigger with your expansion body.

Stash does not log, store, or transmit what you type. No keystroke ever leaves your device. The in-memory window is wiped on focus change, paste, or any non-text-typing event. Password fields are excluded.

What Stash never does

  • No analytics, telemetry, crash reporters, or A/B testing.
  • No browsing-history tracking. We do not see which sites you visit.
  • No content scraping. We do not read page DOM, form fields, cookies, localStorage, or request bodies.
  • No data sale. Ever.
  • No remote code. All JavaScript ships inside the extension package.
  • No ads, advertising IDs, or third-party trackers.

Sensitive data — the Vault

Each new clipboard item is checked locally against rules that recognize things like API keys, JWTs, credit cards, SSNs, and high-entropy secrets.

In every case, a sensitive copy never lands in storage as plaintext:

  • Vault not set up: the copy is dropped and a lock badge flashes on the toolbar. The popup invites you to set up the Vault.
  • Vault set up & unlocked: the secret is encrypted on your device before it's saved.
  • Vault set up & locked: the copy is dropped and a lock badge flashes to alert you to unlock first.

Encryption uses AES-GCM. Your master password is never stored — we derive an encryption key from it using PBKDF2-SHA256 with 600 000 iterations. If you forget your master password, vaulted items cannot be recovered. There is no backdoor.

Optional cross-device sync

Sync is off by default and Stash has no default destination. When you turn it on, you provide a server URL — typically one you run yourself. Vaulted items go over the wire only as ciphertext.

Outgoing network requests, full list

Stash only makes outgoing requests via the optional sync feature, which is off by default and points at a server you provide. There are no other outgoing requests of any kind.

Your controls

  • Pause capture entirely.
  • Disable image capture.
  • Block specific domains.
  • Disable snippet expansion (off by default).
  • Lock the Vault at any time.
  • Export your history as JSON, or wipe everything.

Children

Stash is not directed at children under 13.
